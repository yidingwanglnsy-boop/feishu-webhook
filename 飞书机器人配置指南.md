# 飞书机器人配置指南

本指南将带你一步步在飞书开放平台创建机器人，用于接收消息并调用本地 Claude Code 执行任务。

## 功能概览

- **消息驱动**：在飞书中发送文本消息，机器人自动调用本地 Claude Code 执行并返回结果
- **会话持续**：同一项目自动续接上一次对话，支持多轮交互
- **多项目切换**：通过 `@项目名` 前缀快速切换工作目录
- **安全模式**：`/safe` 前缀运行，工具调用需人工授权
- **图片处理**：发送图片自动保存到本地，可在后续消息中引用路径

---

## 第一步：创建飞书自建应用

1. 打开浏览器，访问 [飞书开放平台](https://open.feishu.cn/)
2. 点击右上角「登录」，使用你的飞书账号登录
3. 登录后，点击「开发者后台」进入应用管理页面
4. 点击「创建自建应用」
5. 填写应用信息：
   - **应用名称**：Claude Code 助手（或你喜欢的名字）
   - **应用描述**：通过飞书消息控制本地 Claude Code 执行任务
   - **应用图标**：上传一个图标（可选）
6. 点击「创建」

---

## 第二步：开启机器人能力

1. 进入刚创建的应用页面
2. 在左侧菜单找到「添加应用能力」
3. 找到「机器人」能力，点击「启用」
4. 启用后，左侧菜单会出现「机器人」选项

---

## 第三步：配置权限

1. 在左侧菜单点击「权限管理」
2. 搜索并开通以下权限：
   - `im:message` - 获取与发送单聊、群组消息
   - `im:message:send_as_bot` - 以应用身份发送消息
   - `im:resource` - 获取消息中的资源文件
   - `im:chat` - 获取群组信息
3. 点击「批量开通」，确认开通所有权限
4. **重要**：权限变更后需要发布新版本才能生效（见第七步）

---

## 第四步：配置事件订阅

这一步让飞书在有新消息时通知你的服务器。

1. 在左侧菜单点击「事件订阅」
2. 选择订阅方式：「将事件发送到请求地址」
3. **请求地址**先空着，等后面启动 ngrok 后再填（见下方说明）
4. 点击「添加事件」，搜索并订阅以下事件：
   - `im.message.receive_v1` - 接收消息
5. 记下页面上的 **Encrypt Key** 和 **Verification Token**（后续配置需要）

---

## 第五步：获取应用凭证

1. 在左侧菜单点击「凭证与基础信息」
2. 记录以下信息（后续配置需要）：
   - **App ID**：形如 `cli_xxxxxxxxxxxx`
   - **App Secret**：点击「显示」后复制

---

## 第六步：配置本地服务 + 内网穿透

由于飞书需要通过公网地址推送事件，你的本地 WSL 服务需要内网穿透。

### 6.1 安装 ngrok

```bash
# 方式一：snap 安装
sudo snap install ngrok

# 方式二：手动下载
# 访问 https://ngrok.com/download 下载 Linux 版本
# 解压后放到 PATH 目录
```

### 6.2 注册 ngrok 账号（免费版即可）

1. 访问 https://dashboard.ngrok.com/signup 注册
2. 登录后，在 https://dashboard.ngrok.com/get-started/your-authtoken 获取 authtoken
3. 在 WSL 中配置：

```bash
ngrok config add-authtoken 你的authtoken
```

### 6.3 启动穿透

```bash
ngrok http 8000
```

启动后会显示类似：

```
Forwarding  https://xxxx-xxxx.ngrok-free.app -> http://localhost:8000
```

**复制这个 https 地址**，这就是你的公网 webhook 地址。

### 6.4 回填请求地址

1. 回到飞书开放平台 → 事件订阅页面
2. 将请求地址填为：`https://xxxx-xxxx.ngrok-free.app/webhook`
3. 点击「添加」，飞书会发送验证请求，如果本地服务已启动会自动通过验证

---

## 第七步：发布应用

1. 在左侧菜单点击「版本管理与发布」
2. 点击「创建版本」
3. 填写版本号（如 1.0.0）和更新说明
4. 点击「保存并发布」
5. 如果你是企业管理员，可以直接审批通过；否则需要管理员审批

---

## 第八步：配置本地 bot

1. 复制环境变量模板：
   ```bash
   cp .env.example .env
   ```
2. 编辑 `.env` 文件，填入第五步获取的 App ID、App Secret 和第四步获取的 Encrypt Key、Verification Token
3. 安装依赖并启动 bot 服务：

```bash
pip install -r requirements.txt
python bot.py
```

4. 另开一个终端启动 ngrok：`ngrok http 8000`
5. 在飞书中搜索你的机器人名称，发一条消息测试

### 配置项说明

所有配置项均支持环境变量覆盖，也可直接修改 `config.py` 中的默认值：

| 配置项 | 环境变量 | 默认值 | 说明 |
|--------|----------|--------|------|
| APP_ID | `FEISHU_APP_ID` | - | 飞书应用 App ID |
| APP_SECRET | `FEISHU_APP_SECRET` | - | 飞书应用 App Secret |
| ENCRYPT_KEY | `FEISHU_ENCRYPT_KEY` | - | 事件订阅 Encrypt Key |
| VERIFICATION_TOKEN | `FEISHU_VERIFICATION_TOKEN` | - | 事件订阅 Verification Token |
| CLAUDE_PROJECT_DIR | `CLAUDE_PROJECT_DIR` | 项目所在目录 | 默认 Claude Code 工作目录 |
| CLAUDE_MAX_TURNS | `CLAUDE_MAX_TURNS` | `20` | Claude Code 最大执行轮次 |
| ALLOWED_USERS | `FEISHU_ALLOWED_USERS` | 空（所有人） | 允许使用的用户 open_id，逗号分隔 |
| PORT | `FEISHU_BOT_PORT` | `8000` | 服务端口 |
| SESSION_TTL_HOURS | `SESSION_TTL_HOURS` | `24` | 会话过期时间（小时） |
| SESSION_MAX_MESSAGES | `SESSION_MAX_MESSAGES` | `50` | 每个会话最大消息数 |

**PROJECT_MAP**：项目名到目录的映射，用于 `@项目名` 切换，在 `config.py` 中直接编辑字典：

```python
PROJECT_MAP = {
    "feishu": "/home/wyd/daily/feishu",
    "stock": "/home/wyd/daily/stock",
    # 添加你自己的项目...
}
```

## 使用说明

### 聊天命令

| 命令 | 说明 |
|------|------|
| 直接发消息 | 以自动模式调用 Claude Code 执行 |
| `/safe 提示词` | 以安全模式运行，工具调用需人工授权 |
| `/new` | 清除所有项目的会话，下次消息开始新对话 |
| `/new @项目名` | 清除指定项目的会话 |
| `/session` | 查看当前活跃会话信息 |

### 多项目切换

在消息前加 `@项目名` 可切换 Claude Code 的工作目录：

```
@feishu 帮我改 bot.py
@stock 分析今天的行情
```

当前可用的项目名：`feishu`、`stock`、`work`、`employ`、`cui`、`toeic`、`cd-wiki`、`llm-wiki`

不加 `@项目名` 时默认使用 `CLAUDE_PROJECT_DIR` 目录。

### 自动模式 vs 安全模式

- **自动模式**（默认）：Claude Code 自动批准所有工具调用（读写文件、执行命令等），适合信任的操作
- **安全模式**（`/safe` 前缀）：Claude Code 执行工具调用前需要授权，被拒绝的操作会反馈给你

```
帮我重构这个函数          → 自动模式，直接执行
/safe 帮我重构这个函数    → 安全模式，操作需授权
```

### 图片处理

直接给机器人发送图片，会自动下载保存到本地并返回文件路径。然后在后续文本消息中引用该路径：

```
分析图片 /tmp/feishu_images/img_xxx.jpg
```

### 会话管理

- 每个用户在每个项目下自动维护一个会话，消息会续接上一次对话
- 会话在超过 TTL（默认 24 小时）或消息数达到上限（默认 50 条）后自动过期
- 执行超过 30 秒时，机器人会发送进度通知
- 回复过长时自动分段发送

## 日常使用

启动顺序：

```bash
# 终端1：启动 ngrok
ngrok http 8000

# 终端2：启动 bot 服务
python bot.py
```

**注意**：ngrok 免费版每次重启会分配新的域名，需要回飞书开放平台更新事件订阅地址。

## 常见问题

### Q: 飞书验证 webhook 失败？
确保本地 bot 服务已启动，ngrok 已在运行，且请求地址格式为 `https://xxx.ngrok-free.app/webhook`。

### Q: 发消息机器人没反应？
1. 检查 bot 服务是否在运行
2. 检查 ngrok 是否在运行
3. 检查飞书开放平台的事件订阅地址是否正确
4. 检查应用是否已发布且审批通过
5. 检查 `config.py` 中的用户白名单是否包含你的 open_id

### Q: ngrok 域名每次都变怎么办？
- 免费版确实会变，可以考虑购买 ngrok 付费版固定域名
- 或者使用其他内网穿透工具：frp、cloudflare tunnel 等

### Q: 如何获取我的 open_id？
启动 bot 服务后，在飞书中给机器人发任意消息，bot 的日志中会打印发送者的 open_id。

### Q: 安全模式下操作被拒绝怎么办？
安全模式下被拒绝的工具调用会列出在回复中。如果需要自动执行，去掉 `/safe` 前缀以自动模式发送即可。

### Q: 如何清除会话重新开始？
发送 `/new` 清除所有项目会话，或 `/new @项目名` 只清除指定项目。发送 `/session` 可查看当前会话状态（消息数、空闲时间等），判断是否需要清除。
